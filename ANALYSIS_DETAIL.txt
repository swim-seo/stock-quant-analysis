1. 시스템 개요
================================================================================

  [목적]
  매일 유튜브, 뉴스, 시장 동향을 일일이 확인하는 시간을 줄이고,
  정보를 자동 수집 → AI 분석 → 한 페이지 브리핑으로 정리하는 시스템.

  [기술 스택]
  - Frontend: Next.js 16 + TypeScript + Tailwind CSS (Vercel 배포)
  - Backend: Python (yfinance, XGBoost, LightGBM)
  - Database: Supabase (PostgreSQL)
  - Vector DB: ChromaDB + KR-SBERT
  - AI: Claude API (claude-sonnet-4-6)
  - 자동화: Railway Cron (월~금 07:00/16:00 KST)
  - 배포: Vercel (웹) + Railway (수집기)
  - 참고: 기존 Streamlit UI는 _legacy/로 이관, Vercel로 완전 전환

  [데이터 흐름]
  수집 (뉴스/유튜브/수급) → Supabase 저장 → Claude 분석 → 웹 표시
  yfinance (시장데이터) → 기술지표 계산 → ML 예측 → 웹 표시

  [자동 스케줄 (Railway Cron)]
  cron: 0 22,7 * * 1-5 (UTC) = 월~금 07:00/16:00 KST

  07:00 KST (장 시작 전):
    1. 뉴스 수집 (10개 종목, 네이버 금융 API)
    2. 외국인/기관 수급 수집
    3. Claude 뉴스 분석 (호재/악재/중립)
    4. 유튜브 오전 수집 (마켓브리핑, 투자의눈)
    5. 아침 브리핑 생성 → 출근길에 /briefing 페이지로 확인

  16:00 KST (장 마감 후):
    1. 뉴스 수집 (장중 뉴스 반영)
    2. 수급 확정 데이터 수집 (외국인/기관 당일 확정치)
    3. Claude 뉴스 분석
    4. 유튜브 오후 수집 (성공투자오후증시)
    5. 저녁 브리핑 생성

3. 기술 지표 (Technical Indicators)
================================================================================

  [기본 지표 - v1]
  - MA5 / MA20 / MA60: 이동평균선 (5일, 20일, 60일)
    → 골든크로스(MA5>MA20 상향돌파), 데드크로스(MA5<MA20 하향돌파) 신호
  - RSI (14일): 상대강도지수
    → 30 이하 과매도(매수), 70 이상 과매수(매도)
  - MACD (12, 26, 9): 이동평균 수렴확산
    → MACD선이 시그널선 위로 교차하면 매수, 아래로 교차하면 매도
  - 볼린저밴드 (20일, 2σ): 가격 변동 범위
    → 상단 돌파=과매수, 하단 이탈=과매도, 밴드폭(BB_width)=변동성

  [신규 지표 - v2]
  - ATR (14일): Average True Range - 변동성 측정
    → 값이 클수록 변동성 큰 상태, ATR_pct = 종가 대비 ATR %
  - Stochastic (14, 3): 스토캐스틱 오실레이터
    → %K > 80 과매수, %K < 20 과매도, %K가 %D 상향돌파 = 매수
  - OBV: On Balance Volume - 거래량 누적
    → 가격 상승시 거래량 더하고, 하락시 빼서 매집/분산 파악
    → OBV > OBV_MA20이면 매집 중
  - ADX (14일): Average Directional Index - 추세 강도
    → ADX > 25 강한 추세, ADX < 20 약한 추세(횡보)
    → +DI > -DI 상승 추세, +DI < -DI 하락 추세
  - CCI (20일): Commodity Channel Index
    → CCI > 100 과매수, CCI < -100 과매도
  - Williams %R (14일): 모멘텀 오실레이터
    → %R > -20 과매수, %R < -80 과매도


4. ML 모델 구성
================================================================================

  [모델 아키텍처: 앙상블 (XGBoost + LightGBM)]

  XGBoost (Gradient Boosting)
  - n_estimators: 300
  - max_depth: 4
  - learning_rate: 0.03
  - subsample: 0.8
  - colsample_bytree: 0.7
  - reg_alpha: 0.1 (L1 정규화)
  - reg_lambda: 1.0 (L2 정규화)

  LightGBM (Gradient Boosting)
  - 동일 하이퍼파라미터
  - leaf-wise 분할 (XGBoost는 level-wise)
  - 더 빠른 학습, 카테고리 피처에 강함

  앙상블 방식: 가중 평균
  - 최종 확률 = 0.5 × XGBoost 확률 + 0.5 × LightGBM 확률
  - 두 모델의 약점을 상호 보완


  [피처 목록 (총 31개)]

  기술 지표 원값 (11개):
    RSI, MACD, MACD_signal, MACD_hist, BB_width,
    ADX, STOCH_K, STOCH_D, CCI, WILLIAMS_R, ATR_pct

  수익률 (5개):
    return_1d (1일), return_3d (3일), return_5d (5일),
    return_10d (10일), return_20d (20일)

  거래량 (2개):
    volume_change (전일 대비 변화율),
    volume_ratio (5일 평균 대비 비율)

  이동평균 대비 위치 (3개):
    price_to_ma5, price_to_ma20, price_to_ma60

  파생 피처 (10개):
    bb_position    : 볼린저밴드 내 위치 (0=하단, 1=상단)
    macd_hist_change: MACD 히스토그램 변화 방향
    candle_strength : 당일 캔들 강도 (종가 위치)
    stoch_kd_diff  : 스토캐스틱 K-D 차이 (모멘텀 방향)
    di_diff        : +DI - -DI (추세 방향)
    atr_change     : ATR 5일 변화율 (변동성 확대/축소)
    obv_momentum   : OBV 10일 변화율 (매집/분산)
    rsi_change     : RSI 3일 변화 속도
    volatility_5d  : 5일 수익률 표준편차
    volatility_20d : 20일 수익률 표준편차
    intraday_range : 일중 변동폭 (고가-저가)/종가
    gap            : 갭 (전일종가 대비 시가 변화)


  [학습/평가 방법]

  1) TimeSeriesSplit 5-Fold 교차검증
     - 일반 KFold와 달리 시간 순서를 유지
     - 미래 데이터로 과거를 예측하는 look-ahead bias 방지
     - Fold 1: train[0:N] → val[N:N+M]
     - Fold 5: train[0:N+4M] → val[N+4M:N+5M]

  2) 80:20 Train/Test Split
     - 앞 80%로 학습, 뒤 20%로 최종 평가
     - 시간 순서 유지 (셔플 안 함)

  3) 타겟 변수
     - target = 1 (다음날 종가 > 오늘 종가 → 상승)
     - target = 0 (다음날 종가 ≤ 오늘 종가 → 하락)


5. 다중 시간프레임 분석 (Multi-Timeframe)
================================================================================

  일봉 + 주봉 결합:
  - 주봉 추세: 주봉 MA5 > MA20 → 상승, MA5 < MA20 → 하락
  - 신호 규칙:
    · 주봉 상승 + 일봉 매수 신호 → 매수 (확정)
    · 주봉 하락 → 일봉 매수 신호 무시 (큰 흐름 역행 방지)
    · 일봉 매도 신호 → 무조건 매도

  → 큰 추세에 역행하는 진입을 차단하여 손실 감소


6. 진입 신호 판단 (Entry Signal)
================================================================================

  5가지 조건을 3단계로 평가:

  ① 정배열: MA5 > MA20 > MA60
     ✅ Pass: 완전 정배열
     ⚠️ Warn: 부분 정배열 (2개만 맞음)
     ❌ Fail: 역배열

  ② 골든크로스: MA5가 MA20을 상향 돌파
     ✅ Pass: 최근 10일 내 발생
     ⚠️ Warn: 11~20일 전 발생
     ❌ Fail: 미발생

  ③ RSI: 현재 RSI 값
     ✅ Pass: 40~60 (적정)
     ⚠️ Warn: 30~40 또는 60~70 (경계)
     ❌ Fail: <30 또는 >70 (과매도/과매수)

  ④ 주봉 추세: 4주 MA vs 12주 MA
     ✅ Pass: 상승 (>+2%)
     ⚠️ Warn: 횡보 (-2%~+2%)
     ❌ Fail: 하락 (<-2%)

  ⑤ 거래량: 오늘 거래량 / 20일 평균
     ✅ Pass: 1.2x 이상
     ⚠️ Warn: 0.8x~1.2x
     ❌ Fail: 0.8x 미만

  최종 판단 (warn=0.5점):
  - 🟢 진입 추천: 총점 ≥ 4
  - 🟡 대기:      총점 2.5~3.5
  - 🔴 위험:      총점 < 2.5

8. 한국판 공포탐욕 지수 (Korea Fear & Greed Index)
================================================================================

  5가지 요소, 각 최대 20점, 총 100점:

  ① 코스피 변동성 (20점)
     - 20일 연환산 변동성 기반
     - 변동성 낮을수록 탐욕 (높은 점수)
     - 범위: 10% 이하=극탐욕(20점), 30% 이상=극공포(0점)

  ② 코스피 모멘텀 (20점)
     - 현재가 vs 20일 이평선 위치
     - 이평선 위면 탐욕, 아래면 공포
     - 범위: +5% 이상=극탐욕, -5% 이하=극공포

  ③ 거래량 모멘텀 (20점)
     - KODEX 200 ETF 오늘 거래량 vs 20일 평균
     - 범위: 2.0배 이상=극탐욕, 0.5배 이하=극공포

  ④ 미국 CNN Fear & Greed 연동 (20점)
     - CNN Fear & Greed Index 실시간 점수
     - 0~100 → 0~20으로 환산

  ⑤ 유튜브 심리 점수 (20점)
     - 최근 20개 영상의 market_sentiment 평균
     - 긍정=+1, 중립=0, 부정=-1
     - -1~+1 → 0~20으로 환산

  총점 해석:
  - 81~100: 극도의 탐욕
  - 61~80:  탐욕
  - 41~60:  중립
  - 21~40:  공포
  - 0~20:   극도의 공포

  웹 표시: 대시보드 좌측에 게이지 바 + 5개 구성요소 바 표시
  탭 전환으로 "섹터별 심리" 확인 가능

  [수집 대상 - 채널 전체 (과거 데이터 backfill용)]
  - 삼프로TV (@3protv): 경제/투자 전반 (~370건/1년, 주식 관련 필터 후)
  - 소수몽키 (@sosumonkey): 미국/한국 주식 (~288건/1년)
  - 머니투데이 (@moneytoday): 시장 분석 (~93건/1년)
  - 슈카월드 (@syukaworld): 경제/시사 (~91건/1년)

  [주식 관련 영상 필터링 (전처리)]
  슈카월드, 삼프로TV 등은 주식 외 콘텐츠도 많으므로 2단계 필터링:
  1차: 제목 키워드 필터 (70개+ 키워드: 주식, 코스피, 반도체, 매수 등)
      → 비관련 영상 사전 제거 (슈카월드 기준 ~82% 필터링)
  2차: Claude API 분석 시 투자 관련 내용 자동 판별

  [과거 데이터 수집 (backfill)]
  명령어: python youtube_collector.py backfill --days 365
  - 4개 채널에서 1년치 영상 스캔
  - 주식 관련 영상만 필터링 후 수집
  - 총 ~842건 수집 대상 (API 비용 ~$28)
  - scan 명령어로 수집 전 건수 미리 확인 가능

  [수집 파이프라인]
  1) yt-dlp로 재생목록/채널에서 영상 가져오기
  2) 주식 관련 영상 1차 필터링 (제목 키워드)
  3) 자막 다운로드 (yt-dlp → youtube_transcript_api 폴백)
  4) Claude API (claude-sonnet-4-6)로 분석:
     - 3줄 요약
     - 시장 감성 (긍정/중립/부정)
     - 언급 종목, 섹터
     - 투자 신호, 리스크 요인
     - 매매 유형 (단타/스윙/장기)
     - 시급성 (오늘/이번주/장기)
  5) Supabase + ChromaDB 저장

  [검색 방식: Hybrid RAG]
  - 벡터 검색: ChromaDB + KR-SBERT 임베딩
  - 키워드 검색: Supabase PostgreSQL
  - RRF (Reciprocal Rank Fusion)로 두 결과 합산

  [웹 표시]
  - 대시보드: 카드형 그리드 (썸네일 + 제목, 클릭 시 요약 펼침)
  - 종목 페이지: 해당 종목 관련 인사이트 3개 (접기/펼치기)
  - 검색 결과: 종목명 검색 → 관련 영상 카드 목록
    · 색깔 점(🟢🟡🔴)으로 시장 전망 한눈에 파악
    · 매매유형(단타/스윙/장기)으로 투자 스타일 매칭
    · 함께 언급된 종목 태그로 섹터 흐름 파악
    · 투자 시그널로 실제 액션 판단

13. 아침 브리핑 자동 생성
================================================================================

  [입력 데이터]
  - 전일 코스피/코스닥 종가 + 등락률 (yfinance)
  - 최근 유튜브 인사이트 20개 (Supabase)
  - 최근 종목 뉴스 10개 + 수급 (Supabase)

  [Claude 브리핑 생성]
  위 데이터를 종합하여 Claude가 다음 항목을 JSON으로 생성:

  1. market_summary: 전일 시장 흐름 + 오늘 전망 (5~8줄)
  2. top_stocks: 오늘 주목할 종목 3~5개
     - 종목명, 주목 이유, 신호 (매수관심/관망/주의)
  3. sector_outlook: 주목 섹터 3~5개
     - 섹터명, 전망 (긍정/중립/부정), 이유
  4. expert_consensus: 유튜브 전문가 종합 의견 (3~4줄)
  5. risk_alerts: 오늘 주의할 리스크 1~3개

  [Supabase 저장]
  테이블: morning_briefing
  - briefing_date (UNIQUE): 날짜별 1개
  - market_summary, top_stocks, sector_outlook 등
  - investor_flow: 종목별 5일 수급 요약

  [웹 표시]
  - /briefing 전용 페이지
  - 대시보드에서 "오늘의 브리핑" 버튼으로 접근
  - 주목 종목 클릭 시 해당 종목 분석 페이지로 이동
