# 한국 주식 AI 분석 시스템 — 프로젝트 개발 히스토리

> 개발 기간: 2025년 4월  
> 스택: Python · Next.js · Claude API · Supabase · Railway · Vercel  
> 목적: 매일 유튜브·뉴스를 수동으로 확인하는 시간을 줄이고, AI가 자동 수집·분석한 정보를 한 페이지 브리핑으로 전달하는 시스템

## V1 — 기술 분석 기반 종목 분석기 (Streamlit)

### 정의
주식 투자 시 "지금 이 종목이 진입 타이밍인가 아닌가"를 판단하는 데 객관적 기준이 없었다.  
차트를 눈으로 보며 감으로 판단하던 것을 수치로 정량화하고 싶었다.

### 문제 상황
- 여러 종목을 동시에 볼 때 각 종목마다 차트 앱을 열고 지표를 수동으로 확인해야 했다.
- RSI, 이동평균선, 거래량 등 여러 지표를 동시에 보는 전용 도구가 없었다.
- "지금 살 타이밍인지" 한마디로 알고 싶은데, 지표마다 말이 달랐다.

### 해결 방법

**캔들스틱 차트 (Canvas 직접 구현)**
- yfinance로 OHLCV 데이터 수집
- Streamlit(app.py) 위에서 HTML Canvas로 캔들스틱 렌더링
- MA5/MA20/MA60 이동평균선, 볼린저밴드 토글
- RSI 서브차트, 거래량 서브차트 (별도 패널)

**진입 신호 판단 — 5가지 조건 3단계 스코어링**
- 정의: 단순 "매수/매도" 2단계 신호는 오류가 많다 → 3단계(✅/⚠️/❌)로 신뢰도 표시
```
① 정배열:    MA5 > MA20 > MA60        — 추세 방향 확인
② 골든크로스: MA5가 MA20 상향 돌파 (10일 내) — 추세 전환 포착
③ RSI:       40~60 구간              — 과매수/과매도 제외
④ 주봉 추세:  주봉 MA5 > MA20         — 상위 타임프레임 게이트
⑤ 거래량:    오늘 거래량 > 20일 평균 × 1.2 — 관심 증가 확인
```
- 최종 판정: 총점 ≥ 4.0 → 🟢 진입 추천 / 2.5~3.5 → 🟡 대기 / < 2.5 → 🔴 위험

**기술 지표 라이브러리 (indicators.py)**
- 12개 지표 직접 구현: MA, RSI, MACD, 볼린저밴드, ATR, 스토캐스틱, OBV, ADX, CCI, Williams%R
- pandas 기반, 순수 Python 연산 (TA-Lib 미사용)

**다중 시간프레임 분석 (multi_timeframe.py)**
- 문제: 일봉에서 골든크로스가 나도 주봉이 하락 추세면 함정일 수 있다.
- 해결: 주봉 MA5 > MA20를 "게이트" 조건으로 적용 — 주봉이 상승 중일 때만 일봉 매수 신호 허용

**배포**: Streamlit Cloud (로컬 수동 실행)

## V2 — AI 수집 파이프라인 + 감성 분석 자동화

### 정의
기술 지표는 과거 가격만 본다. 시장을 실제로 움직이는 건 "뉴스", "전문가 의견", "수급"이다.  
이 정보를 매일 수동으로 보는 건 불가능하므로 AI가 자동으로 수집·분석하게 만들었다.

### 문제 상황
- 유튜브 경제 채널(한국경제TV, 매일경제TV 등)을 매일 20~30개씩 확인 불가
- 뉴스 기사마다 "호재인지 악재인지" 직접 판단해야 했다.
- 외국인/기관 수급 데이터는 네이버 금융 HTML을 수동으로 봐야 했다.
- 정보가 흩어져 있어 "오늘 어떤 섹터가 좋은 상황인가"를 한눈에 볼 수 없었다.

### 해결 방법

**유튜브 자동 수집 + Claude 분석 (youtube_collector.py)**
```
수집 흐름:
1. yt-dlp로 재생목록에서 영상 메타데이터 다운로드
2. youtube-transcript-api로 자막(스크립트) 추출
3. 키워드 필터링 (한국 주식 관련 영상만 선별)
4. Claude API (claude-sonnet-4-6)로 2단계 분석:
   - 1단계: 핵심 내용 추출 (400자 요약)
   - 2단계: 구조화 JSON 추출
     { market_sentiment, key_stocks, key_sectors,
       investment_signals, trading_type, urgency }
5. Supabase youtube_insights 테이블 저장
6. ChromaDB에 KR-SBERT 임베딩 저장 (의미 검색용)
```

- 수집 채널: 한국경제TV, 매일경제TV, 성공투자오후증시 등 6개 재생목록
- 중복 방지: `video_id`로 upsert — 이미 분석한 영상은 재분석 안 함
- 상태 관리: `processed_playlists.json`에 수집 완료 영상 ID 기록

**뉴스 + 수급 수집 (news_collector.py)**
```
1. 네이버 모바일 API: 종목명으로 뉴스 검색
2. HTML 파싱: 네이버 금융 수급 페이지 크롤링
   → 외국인 5일 순매수/순매도, 기관 5일 순매수/순매도
3. Claude API: 뉴스 감성 분석 (호재/악재/중립 + 이유)
4. Supabase stock_news 테이블 저장
```

**아침 브리핑 자동 생성 (morning_briefing.py)**
- 문제: 수집된 데이터가 흩어져 있어 아침에 빠르게 파악하기 어려움
- 해결: Railway 07:00 KST 실행 시 전날 데이터를 종합해 Claude가 브리핑 문서 생성
```
브리핑 구성:
- 시장 전체 전망 (유튜브 감성 집계)
- 섹터별 핫 토픽
- 주목 종목 TOP 3 (근거 포함)
- 리스크 경고
```

**한국판 공포탐욕 지수 (fear_greed_korea.py)**
- 문제: 미국 CNN F&G 지수는 한국 시장을 반영하지 못함
- 해결: 5개 한국 시장 데이터로 직접 계산
```
① 코스피 변동성    — VIX 개념, 20일 변동성 역산 (20점)
② 코스피 모멘텀    — 125일 이동평균 대비 현재 가격 (20점)
③ 거래량 모멘텀    — 20일 평균 거래량 대비 오늘 (20점)
④ 미국 CNN F&G    — 글로벌 투자 심리 반영 (20점)
⑤ 유튜브 감성      — 전문가 14일 평균 감성 (20점)
총합 0~100점: 극도의 공포 / 공포 / 중립 / 탐욕 / 극도의 탐욕
```

**Hybrid RAG 검색 (community_summarizer.py)**
- 문제: "삼성전자 관련 최근 인사이트 찾아줘"를 SQL로 하면 키워드만 매칭됨
- 해결: 두 검색 방법의 결과를 RRF(Reciprocal Rank Fusion)로 통합
```
① 벡터 검색: ChromaDB + KR-SBERT — "의미적으로 비슷한" 영상 찾기
② 키워드 검색: Supabase SQL LIKE — 종목명/채널명 정확 매칭
③ RRF 통합: 두 결과를 순위 기반으로 병합, 상위 N개 반환
```

**Railway 자동화 스케줄 (railway_job.py)**
```
cron: 0 22,7 * * 1-5 (UTC) = 월~금 07:00/16:00 KST

07:00 KST: 뉴스 → 유튜브 오전 수집 → 아침 브리핑 생성
16:00 KST: 뉴스 → 유튜브 오후 수집 → 저녁 브리핑 생성
```

**발생한 기술 문제 & 해결**
| 문제 | 원인 | 해결 |
|------|------|------|
| Claude API 400 에러 | `claude-sonnet-4-6`는 assistant prefill 미지원 | messages 배열에서 assistant 역할 제거 |
| Supabase 설치 오류 | supabase==2.28.3이 pyiceberg 요구 (없음) | `supabase==2.10.0`으로 다운그레이드 |
| ChromaDB FAIL | `./chroma_db` 상대 경로 — 실행 디렉토리 따라 달라짐 | `Path(__file__).parent / "chroma_db"` 절대 경로로 수정 |

## V3 — Next.js 전환 + ML 예측 + 섹터 분석 대시보드

### 정의
Streamlit은 데이터 과학 탐색 도구로는 좋지만, 실제 서비스로 쓰기엔 UI 한계가 있었다.  
모바일에서도 쓸 수 있고, 실시간으로 반응하는 웹 대시보드가 필요했다.

### 문제 상황
- Streamlit UI: 반응형 불가, 커스텀 디자인 어려움, 공유 URL 불안정
- 기술 지표 + AI 인사이트를 한 화면에서 연결해서 보기 힘들었다.
- 섹터별 "지금 어디가 좋은가"를 종합적으로 볼 방법이 없었다.
- ML 예측이 있어도 "왜 이 종목을 사야 하는가"에 대한 AI 설명이 없었다.

### 해결 방법

**Streamlit → Next.js 16 완전 전환**
- Streamlit 코드는 `_legacy/app.py`로 보관
- Next.js App Router + TypeScript + Tailwind CSS
- Vercel 배포: git push → 자동 빌드 → 글로벌 CDN 서비스

**종목 상세 분석 페이지 (/stock)**
- Canvas API로 캔들스틱 차트 직접 구현 (외부 차트 라이브러리 없음)
  - OHLCV + MA선 + 볼린저밴드 + RSI 패널 + 거래량 패널
  - yfinance → Next.js API Route → 브라우저 렌더링
- 진입 신호 5개 조건 카드 (색상별 상태 표시)
- 해당 종목 뉴스 + 수급 (Supabase에서 실시간)
- 관련 YouTube 인사이트 (종목명 매칭)

**ML 예측 앙상블 (ml_model.py)**
```
피처 엔지니어링 (31개 피처):
- 수익률: 1/2/3/5/10/20일
- 이동평균: MA5/20/60
- 지표: RSI, MACD, 볼린저밴드, ATR, Stoch, OBV, ADX, CCI, Williams%R
- 거래량: 5/20일 평균 비율, 변동성

모델:
- XGBoost + LightGBM 앙상블 (50:50 가중 평균)
- TimeSeriesSplit 5-fold 교차검증 (미래 데이터 누출 방지)
- 타겟: 내일 종가 > 오늘 종가 여부 (이진 분류)
- 실제 정확도: 약 51~52% (시장 베이스라인 대비 소폭 우위)
```

**백테스터 (backtester.py / backtest_multi.py)**
- 문제: "이 전략이 실제로 수익이 났을까?" 검증 방법 없음
- 해결: 과거 데이터로 매수/매도 시뮬레이션
```
가정:
- 슬리피지: 0.1% (체결 시 불리한 가격)
- 수수료: 0.15% (매수+매도)
- 전략 A (MTF): 기술 지표 5조건 기반
- 전략 B (ML): XGBoost 예측 기반
결과 (20개 종목, 2년 백테스트):
- MTF 전략: 현대차 +76.7%, 삼성SDI +71.3%
- ML 전략:  한미반도체 +79.8%, 삼성물산 +54.8%
```

**섹터 분석 (sector-signals API)**
- 문제: 종목 하나하나보다 "섹터 로테이션"을 파악해야 큰 흐름을 볼 수 있다.
- 해결: 섹터별 4가지 지표 + YouTube 감성 + 투자자 수급 종합 스코어
```
섹터 스코어 (0~100점):
① RSI 평균 (0~20점): 섹터 내 종목들의 평균 RSI
② MA20 상회 비율 (0~20점): 섹터 내 몇 %가 20일선 위에 있나
③ 5일 모멘텀 (0~20점): 최근 5일 평균 수익률
④ 거래량 비율 (0~20점): 20일 평균 대비 오늘 거래량
⑤ YouTube 감성 (0~20점): 14일 관련 영상 감성

로테이션 단계 분류:
- 상승기 / 진입기 / 과열 / 하락기 / 침체 (5단계)
```

**공포탐욕 게이지 UI**
- 반원형 게이지 (Canvas로 그라데이션 렌더링)
- 0~100 수치와 "극도의 공포 ~ 극도의 탐욕" 5단계 라벨
- 각 구성요소 점수 카드 표시

**발생한 기술 문제 & 해결**
| 문제 | 원인 | 해결 |
|------|------|------|
| yfinance 한국 주식 PBR/PER 없음 | trailingPE, priceToBook이 `.KS/.KQ` 종목에서 null | forwardPE 사용, PBR ≈ ROE×PER (듀퐁 근사) |
| Vercel 배포 오류 | 작업 디렉토리가 stock_analysis/로 이동 후 vercel 실행 | `cd web && vercel --prod`로 정확한 경로 지정 |

## V4 — KRX 전체 종목 + 저평가 스크리닝 + 가상투자 시뮬레이션 + 테마 검색

### 정의
지금까지는 미리 정해놓은 50~60개 종목만 분석했다.  
"내가 모르는 저평가된 좋은 종목을 자동으로 발굴"하고,  
"내 전략이 1년간 얼마를 벌었을지" 직접 시뮬레이션해보고 싶었다.

### 문제 상황 1: 종목 마스터가 빈약함
- 기존 stocks.ts에 50~60개 종목만 수동 입력 → 방산·조선·원전 관련 중소형주 다수 누락
- "이 종목 분석해줘" 했을 때 섹터 분류가 없어 섹터 신호에 반영 안 됨

### 해결 방법 1: KRX 전체 종목 수집 (krx_fetcher.py)
```
수집 방법:
- kind.krx.co.kr HTML 다운로드 (EUC-KR 인코딩 파싱)
- searchType=13 → KOSPI (.KS suffix)
- searchType=14 → KOSDAQ (.KQ suffix)
- 6자리 숫자 코드만 필터링 (우선주, ETF 제외)
- KRX/KSIC 업종명 → 서비스 섹터 매핑 (60개 키워드)

결과: 4,550개 종목 → Supabase stock_master 테이블 저장
섹터 분포: 소재/산업재 1,528 | IT/플랫폼 976 | 금융 204 | 반도체 275 | 바이오 242 ...
```

- 발생 문제: finance-datareader 설치 실패 (pandas 3.0.2 + 32bit Python = Meson 빌드 필요)
- 해결: Python 내장 `urllib` + `html.parser`만으로 재구현 (외부 패키지 0개)

**stocks.ts 대폭 확장 (50개 → 180개)**
- 3개 신규 섹터 추가: 화장품, 로봇, 광통신
- 기존 오류 수정:
  - `460850.KQ` 알테오젠 → `196170.KQ` (야후 파이낸스 데이터 조회 실패하던 것 수정)
  - `383220.KQ` 에코프로 → `086520.KQ`
  - `099550.KQ` 쎄트렉아이 → `099320.KQ`
  - `051600.KS` "한전기술" → "한전KPS"로 이름 수정 + `052690.KS` 한전기술 신규 추가

### 문제 상황 2: 홈 화면의 "종목 언급 빈도 TOP 10"이 무의미
- 유명 대형주(삼성전자, SK하이닉스)는 항상 1~2위 → 새로운 정보가 없음
- "내가 직접 찾아야 아는 저평가된 좋은 종목"을 자동으로 보여주고 싶었다.

### 해결 방법 2: 저평가 우량주 스크리닝 (/api/undervalued)
```
스코어링 기준 (총 100점):
① PBR < 0.5 → +30점 / < 1.0 → +20점 (청산가치 대비 저평가)
② ROE > 20% → +25점 / > 15% → +18점 (자본 효율성)
③ PER < 8  → +20점 / < 12 → +13점 (이익 대비 저렴)
④ 배당수익률 ≥ 4% → +15점 / ≥ 2.5% → +8점
⑤ 부채비율 < 30% → +10점 / > 300% → -10점
⑥ FCF 양수 → +5점

한국 주식 특이사항:
- Yahoo Finance에서 .KS/.KQ 종목의 trailingPE, priceToBook이 null
- 해결: forwardPE 사용, PBR ≈ ROE/100 × PER (듀퐁 관계)

결과: 전체 비ETF 종목 스캔 → 스코어 상위 10개 표시
```

### 문제 상황 3: 백테스트 시작일 고정으로 검증 의미 없음
- 기존 `START_DATE = "2026-04-17"` — 어제부터 오늘까지라 결과가 ±0%
- 실제 전략 검증을 위해 1년 이상 히스토리가 필요했다.

### 해결 방법 3: 가상투자 시뮬레이션 고도화 (/api/portfolio)
```
변경 내용:
- START_DATE: "2026-04-17" → "2025-01-01" (16개월 백테스트)
- YouTube upload_date 매핑 버그 수정:
  - 기존: processed_at (ISO 형식) 사용 → 분석한 날짜로 감성 매핑
  - 수정: upload_date (YYYYMMDD 형식) 사용 → 실제 업로드 날짜로 매핑
  - 변환 로직: 8자리 → YYYY-MM-DD / ISO → 날짜만 추출

3가지 전략 동시 시뮬레이션:
  단기: MA3/10/20, RSI 38~58, 최대 3종목
  중기: MA5/20/60, RSI 35~65, 최대 4종목
  장기: MA10/50/120, 배당 중시, 최대 5종목

시작일 날짜 선택기 추가:
  - 헤더에 <input type="date"> 추가
  - /api/portfolio?startDate=YYYY-MM-DD 파라미터로 동적 조회
```

### 문제 상황 4: 검색 페이지에서 180개 종목을 보는 방법이 없었다
- 종목 검색창만 있어 "어떤 종목이 있는지" 탐색이 불가능
- 방산주·조선주 등 테마별로 묶어서 보고 싶었다.

### 해결 방법 4: 테마 카드 그리드 검색 페이지 (/search)
```
UI 구조:
  상단: "관심 테마" 섹션 — 2열 카드 그리드
    각 카드: 섹터 아이콘 + 이름 + 종목 수 + 미리보기 3개
    클릭: 카드가 전체 폭으로 확장 → 종목 칩 표시 → 클릭 시 /stock 이동
  하단: "KRX 전체 종목 펼치기" 버튼 (클릭 시 on-demand 로드)

설계 원칙:
  - 자주 보는 180개 종목은 항상 즉시 표시 (정적 데이터, JS 번들 포함)
  - 4,500개 KRX 전체는 클릭 시에만 API 호출 (초기 로딩 속도 유지)
```

## 핵심 설계 결정 및 이유

### 1. "No Backend API" 구조
- Supabase가 Python(쓰기)과 Next.js(읽기) 사이의 유일한 연결
- 별도 API 서버 없음 → 인프라 단순화, Vercel 비용 절감
- 단점: Python 컬럼과 TypeScript 타입을 별도로 관리해야 함

### 2. 두 개의 DB (Supabase + ChromaDB)
| | Supabase (PostgreSQL) | ChromaDB |
|--|--|--|
| 저장 | 구조화된 텍스트/수치 | 384차원 부동소수 벡터 |
| 검색 | 날짜·종목명 정확 조건 | 의미 유사도 |
| 용도 | 대시보드, 백테스트 | RAG 유사 검색 |
| 웹 접근 | ✅ Next.js에서 직접 | ❌ Python에서만 |

### 3. YouTube 분석 2단계 Claude 호출
- 1단계에서 요약 → 2단계에서 JSON 구조화 (한 번에 하려면 JSON 파싱 실패율 높음)
- Claude prefill 방식은 `claude-sonnet-4-6`에서 지원 안 됨 → 일반 user 메시지로 변경

### 4. 섹터 로테이션 5단계 분류
- 단순 "좋다/나쁘다"보다 "지금 어느 단계인가"가 더 actionable
- 진입기: 바닥에서 올라오는 중 → 지금이 진입 타이밍
- 과열: 이미 많이 오른 상태 → 신규 매수 위험
